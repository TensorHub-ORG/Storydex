# Storydex 指令仓库

本目录保存 Storydex 内置的通用小说创作指令模板。桌面端通过 `/api/v1/help/prompts` 读取这些 Markdown 文件，并在左侧活动栏的“指令仓库”中展示。

## 文件约定

- 使用一级标题作为指令名称。
- 使用紧随标题的引用段落 `>` 作为简短说明。
- 按用途建立一级分类目录，例如“项目包装”“角色创作”“世界观”“剧情设计”“编辑审校”。
- 可复制执行的正文必须放在第一个 `prompt` 代码块中。
- 可替换参数使用方括号，例如 `[主题]`、`[目标字数]`；程序会自动识别并展示。
- 指令必须适用于任意 Storydex 小说项目，不得写入固定角色名、固定世界观或项目专属路径。
- 默认要求 Agent 先读取当前项目证据，不把推测写成既定事实，也不在未经授权时直接覆盖正文。

## 新增模板示例

````markdown
# 指令标题

> 一句话说明该指令的用途。

## 可直接使用的指令

```prompt
请基于当前 Storydex 小说项目完成以下任务……
可选参数：[参数]
```
````
