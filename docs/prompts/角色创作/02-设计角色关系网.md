# 设计或补全角色关系网

> 围绕指定角色建立有证据、有张力且能够推动剧情的关系结构。

## 可直接使用的指令

```prompt
请基于当前 Storydex 小说项目，为[目标角色，留空则选择主角]设计或补全角色关系网。

要求：
1. 先读取现有角色档案、正文、WIKI 和知识图谱，保留已经发生且有证据支持的关系。
2. 对每条关系写明：关系对象、表层关系、真实诉求、权力差、信任度、冲突点、共同利益、隐藏信息、可变化方向和证据来源。
3. 新建议关系必须标记为“候选”，不得覆盖项目既有事实。
4. 至少覆盖亲密关系、利益关系、对立关系和信息不对称中的 3 类，但不要为了凑数引入无功能角色。
5. 输出关系表、关系变化触发条件、可用于后续章节的 5 个互动场景钩子，以及需要确认的矛盾点。
6. 只输出关系设计；只有用户明确要求时才更新角色文件或知识图谱。

关系密度：[稀疏/适中/复杂]
剧情阶段：[当前阶段或章节范围]
```
