# 根据当前小说项目生成简介

> 读取当前项目的正文、角色、世界观和大纲，生成忠于已有内容的小说简介。

## 可直接使用的指令

```prompt
请基于当前 Storydex 小说项目生成一份可用于作品详情页的小说简介。

要求：
1. 先读取项目概览、现有章节、角色档案、世界书和剧本/大纲，只采用能够被项目文件支持的信息。
2. 提炼主角身份、核心欲望、主要矛盾、世界观特色和故事悬念，但不要提前泄露关键反转、结局或尚未公开的秘密。
3. 不要把推测、候选方案或临时草稿写成既定事实；资料冲突时以正文和用户确认内容为准，并在结果后简短标注冲突。
4. 文风应具有吸引力，但避免空泛夸张、营销套话和角色名单式堆砌。
5. 输出以下内容：
   - 一句话卖点（30 字以内）
   - 短简介（[短简介字数，默认120-180字]）
   - 长简介（[长简介字数，默认300-500字]）
   - 3 个核心看点
6. 只输出简介方案，不修改项目文件。

目标读者：[目标读者，可留空]
侧重方向：[剧情/角色/世界观/情感，可留空]
```
