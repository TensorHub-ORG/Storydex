# 预设管理详细说明

预设管理用于管理 Storydex 的创作规则，例如文风、剧情节奏、对话习惯、人物行动约束、禁用内容、采样参数和临时规则。

预设是文件化管理的。用户可以通过界面导入、启用、停用和编辑，也可以在项目目录中人工查看。

## 入口

1. 打开 Storydex 小说项目。
2. 点击左侧活动栏的预设管理图标。
3. 打开 `预设管理` 面板。

## 预设目录

预设通常位于：

```text
.storydex/presets/
```

常见子目录：

```text
.storydex/presets/active/       已启用预设
.storydex/presets/library/      未启用预设库
.storydex/presets/blocked/      禁用预设
.storydex/presets/compiled/     编译后的预设
```

只有 `active` 或编译后的预设可以进入本轮生成上下文。

## 界面区域

| 区域 | 说明 |
| --- | --- |
| 顶部标题 `预设管理` | 当前模块名称。 |
| 项目名称 | 当前小说项目名称。 |
| `目录 .storydex/presets` | 预设文件所在目录。 |
| 导入工作台 | 用于导入外部预设。 |
| `已启用` | 当前会影响生成上下文的预设。 |
| `未启用` | 已保存但不会注入上下文的预设。 |

## 顶部按钮

| 图标/按钮 | 标题 | 作用 |
| --- | --- | --- |
| 导入图标 | `导入` | 选择外部预设文件并进入导入预览。 |
| 刷新图标 | `刷新` | 重新读取预设目录。 |

## 导入工作台

| 元素 | 作用 |
| --- | --- |
| `导入` 按钮 | 打开文件选择器，选择要导入的预设。 |

导入后会进入 `导入预览`，不是立即启用。

## 导入预览窗口

| 区域/按钮 | 作用 |
| --- | --- |
| `导入预览` 标题 | 查看即将导入的预设内容。 |
| 关闭按钮 `X` | 关闭导入预览。 |
| 模块数量 | 显示预设中识别出的模块数量。 |
| 宏提示 | 显示导入时保留的外部预设专有宏。 |
| `模块列表` | 查看预设包含哪些模块。 |
| `宏提示` | 查看按原文保留、运行时兼容层会尽量展开的宏或特殊语法。 |
| `展示正则` | 查看展示/匹配相关的正则规则。 |
| `SPreset ChatSquash 元数据` | 查看导入文件中携带的元信息。 |
| `取消` | 放弃本次导入。 |
| `确认导入` | 把预设写入 Storydex 项目。 |

建议导入前先看宏提示。外部预设里的专有宏会尽量保留原文，后续运行时兼容层再决定如何展开。

## 预设列表行

| 元素 | 说明 |
| --- | --- |
| 文件图标 | 表示这是一个预设文件。 |
| 预设名称 | 例如 `00 小说人称预设.md`。 |
| 副标题 | 通常显示文件名或路径。 |
| 参数图标 `tune` | 打开该预设的参数编辑器。 |
| 启用/停用开关 | 在 `active` 和 `library` 之间移动预设。 |
| `MD` 等扩展名标记 | 显示文件类型。 |

## 已启用和未启用

| 区域 | 含义 | 对生成的影响 |
| --- | --- | --- |
| `已启用` | 位于 `.storydex/presets/active` 的预设。 | 会被 Context Assembler 视为可注入上下文的来源。 |
| `未启用` | 位于 `.storydex/presets/library` 的预设。 | 保存在项目中，但默认不影响生成。 |

预设不是越多越好。启用太多互相冲突的规则，会让生成变慢或风格不稳定。

## 预设行按钮

| 按钮/图标 | 作用 |
| --- | --- |
| 点击预设行 | 在中间编辑区打开预设文件。 |
| `tune` / `编辑参数` | 打开结构化参数编辑器。 |
| `toggle_on` / `停用` | 把已启用预设移回 library。 |
| `toggle_off` / `激活` | 把未启用预设移动到 active。 |

## 参数编辑器

点击 `编辑参数` 后，会打开预设参数编辑器。

### 顶部按钮

| 按钮 | 作用 |
| --- | --- |
| `保存` | 保存当前预设参数。 |

### 编辑模式

| 模式 | 作用 |
| --- | --- |
| `结构化编辑` | 用表单编辑采样参数、长度约束、风格规则、思考阶段等。 |
| `写作工作台` | 编辑临时写作规则、自由文本、写作偏好等更贴近创作的内容。 |
| `注入预览` | 查看当前预设最终会注入给模型的文本。 |
| `风险体检` | 查看预设是否存在冲突、过长、特殊宏或不兼容内容。 |

### 常见参数

| 参数 | 作用 | 建议 |
| --- | --- | --- |
| `temperature` | 控制随机性。越高越发散。 | 小说可略高，但不要过高。 |
| `top_p` | 控制候选词采样范围。 | 通常保持默认。 |
| `top_k` | 限制候选词数量。 | 不确定时保持默认。 |
| `frequency_penalty` | 减少重复用词。 | 重复明显时小幅提高。 |
| `presence_penalty` | 鼓励引入新内容。 | 剧情停滞时可小幅提高。 |
| `seed` | 随机种子。 | 需要复现实验时填写。 |
| 正文字数下限/目标/上限 | 控制正文片段长度。 | 与 Agent 面板的片段字数保持一致更稳。 |
| 段落长度下限/上限 | 控制段落密度。 | 长篇小说建议避免段落过长。 |
| 思考阶段 | 让 Agent 在生成前按步骤规划。 | 适合复杂剧情、变量更新、WIKI 整理。 |
| 视角/叙述者 | 控制 POV 和叙述风格。 | 与小说主风格保持一致。 |
| 禁用词/禁用模式 | 避免某些词、桥段或表达。 | 不要写得过宽，否则会误伤正常内容。 |
| 风格规则 | 约束文风、对白、描写方式。 | 用自然语言写清楚，不要堆太多互斥要求。 |

## 常用操作

### 导入外部预设

1. 打开 `预设管理`。
2. 点击顶部 `导入`。
3. 选择预设文件。
4. 在 `导入预览` 中查看模块列表和宏提示。
5. 确认可用后点击 `确认导入`。
6. 导入后通常先进入 `未启用`。
7. 需要使用时再点击启用开关。

### 启用预设

1. 在 `未启用` 中找到目标预设。
2. 点击该行右侧的启用开关。
3. 预设会移动到 `已启用`。
4. 下一轮生成时，它才可能进入上下文。

### 停用预设

1. 在 `已启用` 中找到目标预设。
2. 点击停用开关。
3. 预设会移动回 `未启用`。
4. 后续生成不再默认使用它。

### 编辑预设参数

1. 点击预设行右侧 `tune`。
2. 在 `结构化编辑` 中调整参数。
3. 打开 `注入预览` 检查实际注入文本。
4. 打开 `风险体检` 查看问题。
5. 点击 `保存`。

## 预设编写建议

- 写清楚“应该做什么”，少写模糊口号。
- 不要同时启用互相冲突的文风预设。
- 不要把整本 WIKI 或全部角色档案塞进预设。
- 预设应负责规则，项目事实应放在角色、变量、WIKI 和记忆里。
- 对剧情、对话、人物行动约束，可以使用 Markdown 自然语言表达。

## 和变量思考的关系

变量思考不建议做成难维护的硬性 JSON 条目。更推荐用自然语言或 Markdown 写清楚：

- 本轮有哪些状态变化。
- 哪些变化已确定。
- 哪些变化不确定。
- 是否与旧设定冲突。
- 是否建议写入变量文件。

预设可以约束变量思考的风格，但不要强迫用户维护固定 JSON 路径和值。

## 常见问题

### 预设启用了为什么没有生效？

可能原因：

1. 预设没有放在 `active`。
2. 预设太长，被上下文预算压缩或跳过。
3. 当前任务不需要使用该预设。
4. 预设内容和其他规则冲突。
5. 还没有保存或刷新。

### 可以直接编辑预设 Markdown 吗？

可以。预设是文件化管理的。编辑后建议保存，并在预设管理中刷新或查看注入预览。

### blocked 是什么？

`blocked` 用于放置明确不应进入运行时上下文的预设。它可以作为保留区或禁用区。

### compiled preset 是什么？

它是经过整理或编译后适合注入运行时的预设版本，目的是让运行时读取到稳定的预设结构。
