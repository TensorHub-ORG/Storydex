# 目录结构与资源浏览器详细说明

资源浏览器用于打开小说项目、查看 Storydex 项目目录、创建文件/文件夹、打开正文和设定文件，以及通过右键菜单整理项目资源。

Storydex 项目本质上是一个普通本地文件夹。用户可以人工编辑，Agent 也可以在授权后自动管理。

## 入口

1. 打开 Storydex。
2. 点击左侧活动栏的资源图标。
3. 左侧面板会显示 `资源浏览器`。

如果还没有打开项目，会先显示欢迎状态，提供 `打开文件夹` 和 `新建项目`。

## 欢迎状态按钮

| 按钮 | 作用 | 什么时候使用 |
| --- | --- | --- |
| `打开文件夹` | 选择一个已有的本地写作目录。 | 已经有小说项目或想打开旧项目时使用。 |
| `新建项目` | 创建一个新的 Storydex 工作区。 | 第一次写新故事，或想建立独立测试项目时使用。 |

## 资源浏览器顶部区域

| 区域 | 说明 |
| --- | --- |
| `资源浏览器` | 当前面板标题。 |
| 项目名称 | 当前打开的小说项目名。 |
| 项目路径 | 当前小说项目所在的本地路径。 |
| 顶部工具栏 | 提供新建、刷新、保存等快捷操作。 |

## 顶部工具栏按钮

| 图标/按钮 | 标题 | 作用 | 使用方式 |
| --- | --- | --- | --- |
| `note_add` | `新建文件` | 在项目根目录新建文件。 | 点击后输入文件名，例如 `灵感.md`，按 Enter 确认。 |
| `create_new_folder` | `新建文件夹` | 在项目根目录新建文件夹。 | 点击后输入文件夹名，按 Enter 确认。 |
| `refresh` | `刷新` | 重新读取项目目录树。 | 手动改了文件，或 Agent 写入后列表没更新时使用。 |
| `save` | `保存` | 保存当前编辑器中已修改的文件。 | 文件内容有未保存修改时使用。 |

新建或重命名时：

| 操作 | 作用 |
| --- | --- |
| `Enter` | 确认创建或重命名。 |
| `Esc` | 取消创建或重命名。 |

## 常见项目目录

不同模板的项目会有差异，但典型 Storydex 项目会包含：

```text
.coomi/                         Coomi/Agent 运行相关数据
.storydex/                      Storydex 项目配置、模板、记忆、技能、追踪数据
.storydex/.agent/               Agent 技能、规则、记忆、计划和失败记录
.storydex/.agent/skills/        从应用 docs/skills 复制的通用内置技能和项目自定义技能
.storydex/characters/           Storydex 内部角色索引或兼容目录
.storydex/config/               项目配置
.storydex/memory/               项目记忆、章节进度、当前变量等
.storydex/presets/active/       已启用预设
.storydex/presets/blocked/      禁用或阻止进入上下文的预设
.storydex/presets/compiled/     编译后的安全预设
.storydex/presets/library/      未启用预设库
.storydex/scripts/              脚本或剧本类资料
.storydex/temp/                 临时数据
.storydex/templates/            模板文件
.storydex/.agent/sessions/      Agent 会话历史和工作记录
.storydex/wiki/                 WIKI 和知识图谱相关数据
.storydex/worldbook/            世界书或设定资料
chapters/                       正文章节和剧情片段
.gitignore                      Git 忽略规则
README.md                       项目说明
```

用户可以扩展目录，但不要手动删除这些基础目录：

```text
.storydex/
.coomi/
.git/
```

其中 `.git/` 是小说项目的版本控制，不是 Storydex 源码仓库。

## 目录树操作

| 操作 | 结果 |
| --- | --- |
| 单击文件 | 在中间编辑区打开该文件。 |
| 单击文件夹左侧箭头 | 展开或收起文件夹。 |
| 单击章节目录的 `完成` 开关 | 标记该章节目录是否已完成。 |
| 右键文件 | 打开文件操作菜单。 |
| 右键文件夹 | 打开文件夹操作菜单。 |
| 右键空白区域 | 打开根目录操作菜单。 |
| 拖拽文件到文件夹 | 将文件移动到目标文件夹，适合整理目录。 |

## 章节目录的完成开关

章节目录行右侧可能出现 `完成` 开关。

| 状态 | 含义 |
| --- | --- |
| 关闭 | 该章节目录还在写作或未完成。 |
| 开启 | 该章节目录已完成。 |

这个状态会影响 Storydex 对章节进度的理解。建议在一个章节或一组剧情片段稳定后再标记完成。

## 推荐的文件放置方式

| 内容 | 推荐位置 |
| --- | --- |
| 正文、剧情片段 | `chapters/` |
| 角色档案 | `characters/` 或模板指定的角色目录 |
| 世界观、势力、地点 | `worldbook/` 或 `.storydex/worldbook/` |
| WIKI 条目 | `wiki/` 或 `.storydex/wiki/` |
| 创作预设 | `.storydex/presets/` |
| 章节模板 | `.storydex/templates/chapters/` |
| 临时草稿 | `temp/` 或用户自建草稿目录 |

## 常用操作

### 打开已有项目

1. 点击左侧资源图标。
2. 点击 `打开文件夹`。
3. 选择小说项目根目录。
4. 等待资源树加载。
5. 确认 `chapters/`、`.storydex/` 等目录正常显示。

### 新建文件

1. 点击顶部 `新建文件`，或右键目标文件夹选择 `新建文件`。
2. 输入文件名，例如 `第001章.md`。
3. 按 Enter。
4. 点击新文件，在中间编辑区写内容。
5. 点击顶部 `保存`。

### 整理目录

1. 先刷新目录树。
2. 找到放错位置的文件。
3. 用拖拽、剪切/粘贴或重命名整理。
4. 整理后打开版本控制查看更改。
5. 如果是 Agent 整理，执行结束后应自动产生小说项目本地提交。

### 把文件交给 Agent 参考

1. 右键目标文件。
2. 选择 `添加到聊天`。
3. 在 Agent 输入框说明任务，例如“请根据这个角色文件更新最近章节里的称呼一致性”。

## 注意事项

- Storydex 可以管理普通 Markdown、文本和 JSON 文件，但普通用户不需要理解所有内部 JSON。
- `.storydex/` 是项目核心配置区，除非知道用途，不建议手动移动或删除。
- `.storydex/presets/active` 中的预设会影响生成上下文，移动预设前先确认用途。
- 新建项目的内置技能来自应用的 `docs/skills`；用户修改过的项目技能不会被默认模板静默覆盖。
- 章节目录模板决定新故事的组织方式，新故事开始前要慎重选择。
- 手动修改文件后，建议刷新资源树并查看版本控制。


```text

```

```text

```

## 常见问题

### 为什么欢迎页还有历史记录或其他侧栏？

欢迎页表示还没有正式打开项目。此时资源浏览器应只引导打开文件夹或新建项目；如果仍看到项目历史或右侧内容，可能是界面状态没有重置，刷新或重新打开 Storydex 后再试。

### 我可以自己建目录吗？

可以。Storydex 允许项目扩展目录。只要不删除基础骨架，用户可以按自己的创作习惯增加资料夹。

### 为什么有些文件夹是红色或有标记？

通常表示该目录或其子项存在诊断信息、变更状态或需要注意的项目状态。可以点击或右键查看具体文件。

### 删除文件后能恢复吗？

如果小说项目已经初始化版本控制，通常可以通过本地 Git 历史找回旧版本。删除重要文件前仍建议先确认。
