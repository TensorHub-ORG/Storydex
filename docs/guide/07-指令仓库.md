# 指令仓库

指令仓库用于集中管理 Storydex 内置的通用小说创作提示词。模板文件位于源码目录 `docs/prompts/`，桌面版封装时会一并复制，因此开发环境和正式安装包使用同一套内容。

## 打开方式

1. 启动 Storydex。
2. 在左侧活动栏找到“知识图谱”图标。
3. 点击其下方的“指令仓库”图标。

指令仓库不要求先打开小说项目即可浏览；“填入 Agent”功能需要当前已经打开一个 Storydex 小说项目。

## 主要功能

| 功能 | 说明 |
| --- | --- |
| 分类筛选 | 按项目包装、角色创作、世界观、剧情设计、编辑审校等目录筛选。 |
| 关键词搜索 | 同时搜索指令标题、说明、分类和正文。 |
| 参数提示 | 自动识别 `[主题]`、`[目标字数]` 等可替换参数。 |
| 复制指令 | 将完整指令正文复制到剪贴板。 |
| 填入 Agent | 把指令放入右侧 Agent 输入框，替换参数后即可发送。 |
| 来源定位 | 在详情底部显示对应的 `docs/prompts` 文件路径。 |

## 使用建议

1. 先打开需要处理的小说项目。
2. 选择最接近当前任务的指令模板。
3. 将方括号参数替换为实际内容；不需要的可选参数可以删除或写“留空”。
4. 点击“填入 Agent”。
5. 再次检查范围、目标字数、是否允许写入文件等关键要求，然后发送。

内置指令默认要求 Agent 先读取当前项目证据、区分事实与候选设定，并在未获授权时不直接覆盖正文。

## 新增或修改指令

每个指令都是 Markdown 文件。推荐按用途放入 `docs/prompts/<分类>/`，并使用以下格式：

````markdown
# 指令名称

> 一句话说明。

## 可直接使用的指令

```prompt
请基于当前 Storydex 小说项目……
参数：[参数名称]
```
````

程序读取第一个 `prompt` 或 `text` 代码块作为可复制正文。方括号参数会自动显示为参数标签。

## 常见问题

### 看不到新模板

- 确认文件扩展名为 `.md`。
- 确认文件不叫 `README.md`；README 只用于目录说明，不作为指令展示。
- 点击指令仓库右上角刷新按钮。
- 正式桌面版新增模板后需要重新封装，才能进入安装包。

### “填入 Agent”按钮不可用

请先打开或创建一个 Storydex 小说项目。浏览和复制不受影响。
