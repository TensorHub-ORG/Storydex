# WIKI 整理

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜以证据为基础维护 WIKI 和知识图谱。

## 1. 技能用途

生成、更新、审阅或修复项目 WIKI 与知识图谱，使实体、事实、关系、事件和伏笔可检索、可追溯。

## 2. 适用触发

- 用户要求整理 WIKI、知识图谱、实体关系或伏笔。
- 正文或项目资产变化后需要增量同步知识结构。

## 3. 输入检查

- 待处理章节、角色、世界书、剧本和现有 WIKI。
- 实体稳定 ID、别名、证据来源和当前关系图。

## 4. 证据规则

- 正式事实和关系必须有来源路径或剧情证据。
- 显示名称变化不得创建重复实体。
- 推测、候选、冲突和过时内容必须单独标记。

## 5. 允许落点

- `.storydex/wiki/`
- `.storydex/memory/current/relationship_graph.json`，仅通过受控更新

## 6. 执行步骤

1. 读取现有索引，识别相关实体并复用稳定 ID。
2. 从权威来源提取实体、事实、关系、事件和伏笔。
3. 区分新增、更新、冲突、过时、别名和待确认项。
4. 最小增量更新条目和关系，保留证据。
5. 检查孤立、重复、无证据关系和知识边界泄露。

## 7. 输出模板

```markdown
# WIKI 增量整理报告
## 处理范围
## 实体变更
- 新增：
- 更新：
- 别名/合并候选：
## 关系变更
- 主体 → 关系 → 客体
- 证据：
- 置信度：
## 事件与伏笔
## 冲突、过时与待确认
## 建议写入路径
```

## 8. 完成前自检

- [ ] 每个正式事实和关系都有来源。
- [ ] 没有产生重复实体。
- [ ] 推测和冲突没有被写成确定事实。
- [ ] WIKI 没有反向覆盖正文或正式角色设定。

## 9. 安全边界

- 实体合并、删除和重大关系变化先请求确认。
- 不把索引或派生摘要作为权威来源。
- 不读取或修改当前项目之外的知识库。
