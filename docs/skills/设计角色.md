# 设计角色

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜创建或补全可持续使用的角色档案。

## 1. 技能用途

创建与当前项目世界规则、剧情需求和角色网络兼容的新角色，或把角色草案补全为正式档案。

## 2. 适用触发

- 用户要求创建新角色、角色卡、人物弧线或功能角色。
- 剧情需要新的对手、盟友、导师、线索人物或关系节点。

## 3. 输入检查

- 角色定位、剧情功能、首次出场阶段和用户明确约束。
- 现有角色、世界书、组织势力、力量体系、当前剧情和命名习惯。
- 检查能否扩展已有角色，避免无意义重复。

## 4. 证据规则

- 用户本轮要求和正文事实优先。
- 项目未提供的信息写“未知”或“候选”。
- 新建议不得冒充已经发生的剧情事实。

## 5. 允许落点

- 正式档案：`.storydex/characters/`
- 未确认草案：回答正文或 `.storydex/temp/`

## 6. 执行步骤

1. 读取相关项目资产并列出不可违反的约束。
2. 设计身份、叙事功能、目标、需求、恐惧、秘密、能力边界和行为模式。
3. 建立与现有角色、势力、地点或事件的关系钩子。
4. 区分已确认、建议和待确认信息。
5. 按项目角色模板输出；用户确认后再选择不冲突的文件名写入。

## 7. 输出模板

```markdown
# [角色名]
## 定位
- 身份：
- 叙事功能：
- 当前剧情作用：
## 基本信息
- 年龄：未知
- 外貌：未知
- 身份/职业：未知
- 常驻地点：未知
## 性格与行为模式
- 核心性格：
- 说话方式：
- 压力下的反应：
## 关系网络
- [对象]：表层关系、利益、冲突、信任与信息差
## 动机、秘密与边界
- 外在目标：
- 内在需求：
- 核心恐惧：
- 秘密/未知信息：
- 能力边界：
## 角色弧线与出场建议
## 事实状态
- 已确认：
- 本次建议：
- 待确认：
```

## 8. 完成前自检

- [ ] 没有违反时代、世界规则或力量成本。
- [ ] 角色功能不与已有角色无意义重叠。
- [ ] 未知信息已明确标记。
- [ ] 角色没有提前知道其不应知道的信息。

## 9. 安全边界

- 用户只要求草案时不落盘。
- 不覆盖已有角色文件，不删除自定义字段。
- 重大关系或身份设定写入前必须确认。
