# 变量思考

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜分析剧情片段产生的明确状态变化。

## 1. 技能用途

分析剧情造成的时间、地点、角色、关系、物品、事件和世界状态变化，优先生成可读的增量说明。

## 2. 适用触发

- 新剧情生成或正文修改后需要同步状态。
- 用户要求总结变量、检查连续性或生成章节快照。

## 3. 输入检查

- 本次剧情片段及其前一状态。
- 当前变量、角色档案、关系图、物品和时间线。
- 明确处理范围，避免一次分析无关章节。

## 4. 证据规则

- 变化必须来自本次片段或用户确认。
- 变量思考是辅助说明，不自动成为最高优先级正式事实。
- 不确定变化写“待确认”，不能生成确定的机器操作。

## 5. 允许落点

- 可读记录：`.storydex/memory/chapters/`
- 明确结构化状态：`.storydex/memory/current-state/`

## 6. 执行步骤

1. 定位片段前后的状态边界。
2. 按时间地点、角色、关系、物品、事件、线索和环境分类。
3. 区分持续状态、瞬时状态、未知、冲突和待确认内容。
4. 先生成 Markdown；只有确定变化才生成结构化操作。
5. 写入前校验 revision、证据、引用和冲突。

## 7. 输出模板

```markdown
# [章节/片段]变量思考
## 来源
- 文件：
- 时间范围：
## 明确变化
### 时间、地点与环境
### 角色状态与知识
### 关系
### 物品与资源
### 事件、任务与伏笔
## 保持不变但需要关注
## 冲突与待确认
## 可选结构化操作
- path / op / value / evidence / baseRevision
```

## 8. 完成前自检

- [ ] 所有变化都有证据。
- [ ] 变量思考没有覆盖正式事实。
- [ ] 机器操作只包含可安全合并的变化。
- [ ] 删除、冲突和重大关系变化已进入确认。

## 9. 安全边界

- 不保存聊天记录、工具日志或临时推理。
- 不对不确定状态执行结构化写入。
- 不自动删除旧状态和恢复点。
