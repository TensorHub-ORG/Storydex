# 设计剧本

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜规划卷纲、章节、场景和伏笔路线。

## 1. 技能用途

规划卷纲、章节路线、关键场景、冲突升级、人物弧线和伏笔回收，为正文生成提供可审阅路线。

## 2. 适用触发

- 用户要求设计大纲、卷纲、章节计划、场景或剧情分支。
- 当前剧情缺少明确目标、冲突或后续推进路线。

## 3. 输入检查

- 当前剧情起点、篇幅、章节数、节奏和必须发生的事件。
- 正文、未解决线索、角色状态、世界规则、预设和已有剧本。

## 4. 证据规则

- 既有正文和用户确认路线不可被候选大纲反向覆盖。
- 角色只能依据其当前知识做出选择。
- 新事件必须有因果来源，不能只靠巧合推进。

## 5. 允许落点

- 正式剧本：`.storydex/scripts/`
- 未确认草案：回答正文或 `.storydex/temp/`

## 6. 执行步骤

1. 总结当前状态、角色目标、未解决冲突和连续性约束。
2. 确定阶段目标、阻力、升级链、关键选择和阶段终点。
3. 拆分为有目标、冲突、变化和钩子的章节或场景。
4. 建立伏笔埋设、强化、误导和回收表。
5. 输出风险和备选路线，确认后再写入剧本目录。

## 7. 输出模板

```markdown
# [卷名/剧情阶段]剧本
## 起点状态
## 阶段目标与核心冲突
## 人物弧线
- [角色]：起点 → 触发 → 选择 → 阶段结果
## 章节/场景计划
### [章节或场景]
- 目标：
- 冲突：
- 关键行动：
- 信息揭示：
- 状态变化：
- 结尾钩子：
## 伏笔表
- [伏笔]：埋设 / 强化 / 回收
## 连续性风险与备选路线
```

## 8. 完成前自检

- [ ] 每个章节或场景都产生有效变化。
- [ ] 冲突升级具有因果链。
- [ ] 角色知识边界正确。
- [ ] 剧本没有未经授权改写正文。

## 9. 安全边界

- 不直接覆盖正文。
- 不把候选路线写成已经发生的事实。
- 重大剧情方向和结局变化必须由用户确认。
