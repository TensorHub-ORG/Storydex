# 项目目录整理

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜安全整理目录、命名和文件职责。

## 1. 技能用途

在不破坏 Storydex 基础骨架和用户文件的前提下，整理小说项目目录、命名、归类和说明文档。

## 2. 适用触发

- 用户要求整理文件、迁移目录、统一命名或修复结构。
- 项目存在散落文件、重复目录、职责不清或旧结构。

## 3. 输入检查

- 当前完整目录树、项目 Manifest 和用户整理目标。
- 文件引用、打开标签、Git 状态和依赖旧路径的配置。

## 4. 证据规则

- 只读扫描后再提出方案。
- 无法确认用途的文件保持不动并标记待确认。
- Git 修改和用户自定义目录必须优先保护。

## 5. 允许落点

- `.storydex/`
- `chapters/`
- 用户明确允许整理的其他项目内目录

## 6. 执行步骤

1. 识别基础骨架、用户文件、临时文件、重复项和旧结构。
2. 提出保留、移动、新建、合并、归档和不处理清单。
3. 检查引用、命名风格、章节顺序和 Git 修改。
4. 重大移动、合并和潜在删除先请求确认。
5. 小批次执行并复查目录树、引用和项目打开状态。

## 7. 输出模板

```markdown
# 项目目录整理方案
## 当前问题
## 保留不动
## 建议新建
- 路径 / 用途
## 建议移动或重命名
- 原路径 → 新路径 / 理由 / 影响
## 重复与归档候选
## 需要确认的高风险操作
## 执行后验证清单
```

## 8. 完成前自检

- [ ] 基础骨架与自定义目录得到保护。
- [ ] 没有未经确认的删除或覆盖。
- [ ] 移动后引用和配置仍然有效。
- [ ] 没有强迫所有小说使用不必要的额外目录。

## 9. 安全边界

- `.storydex/temp/` 与 `.storydex/.agent/temp/` 不得混用。
- 不执行不可回退的大批量操作。
- 不越过当前小说项目根目录。
