# 角色更新

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜根据正文证据增量同步角色状态。

## 1. 技能用途

根据新正文或用户确认内容，更新角色状态、关系、知识边界和人物弧线，同时保留稳定设定。

## 2. 适用触发

- 新章节改变了角色位置、状态、关系、物品、能力或认知。
- 用户要求同步角色档案、检查人物连续性或补录新角色。

## 3. 输入检查

- 本次变化对应的章节、片段和直接证据。
- 目标角色原档案、相关角色、当前变量和知识图谱。

## 4. 证据规则

- 每项变化必须能够追溯到正文或用户确认。
- 短期情绪不等于稳定性格，临时位置不等于常驻地点。
- 派生摘要不能覆盖正式角色事实。

## 5. 允许落点

- `.storydex/characters/`
- `.storydex/memory/` 中有证据支持的长期状态

## 6. 执行步骤

1. 对比更新前档案和本次剧情证据。
2. 列出新增、修改、保持、冲突和待确认项。
3. 检查知识边界、关系、物品、能力与位置连续性。
4. 以最小差异修改原档案，新角色使用项目角色模板。
5. 保留证据来源和用户自定义内容。

## 7. 输出模板

```markdown
# [角色名]增量更新
## 证据来源
- 文件/章节：
- 关键证据：
## 变更摘要
- 新增：
- 修改：
- 保持：
- 冲突/待确认：
## 状态更新
- 时间与位置：
- 身体与情绪：
- 关系：
- 物品与能力：
- 已知信息：
- 未知或误解：
## 建议写入
- 目标文件：
- 最小修改范围：
```

## 8. 完成前自检

- [ ] 每项变化均有证据。
- [ ] 稳定设定与阶段状态没有混淆。
- [ ] 用户字段和有效历史得到保留。
- [ ] 冲突没有被静默覆盖。

## 9. 安全边界

- 不删除可能仍有效的历史信息。
- 重大身份、关系或生死变化先请求确认。
- 用户只要求分析时不写入文件。
