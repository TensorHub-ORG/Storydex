# 设计世界书条目

> 模板版本：2｜适用范围：任意 Storydex 小说项目｜创建规则明确且服务剧情的世界设定。

## 1. 技能用途

创建或补全地点、势力、物品、制度、历史、能力体系、种族、技术和规则等世界书条目。

## 2. 适用触发

- 用户要求设计世界观、地点、组织、物品、规则或力量体系。
- 正文出现需要沉淀为长期设定的新概念。

## 3. 输入检查

- 条目主题、题材、时代、规模、氛围和用户约束。
- 现有世界书、正文、角色、势力关系和相关 WIKI。
- 检查是新建、扩展还是关联已有条目。

## 4. 证据规则

- 正文和用户确认内容属于已确认设定。
- 本次创作方案属于建议设定，未确认前不得影响后续剧情。
- 推测和矛盾内容必须单独标注。

## 5. 允许落点

- 正式条目：`.storydex/worldbook/`
- 候选方案：回答正文或 `.storydex/temp/`

## 6. 执行步骤

1. 识别条目的剧情功能及与人物选择的关系。
2. 定义核心规则、范围、成本、限制、反制、例外和失败后果。
3. 补充历史、现状、日常影响、关联实体和证据。
4. 检查与现有设定的冲突和重复。
5. 用户确认后按项目命名习惯写入独立文件。

## 8. 完成前自检

- [ ] 规则具有成本、限制和反制。
- [ ] 设定没有与已有内容发生未说明冲突。
- [ ] 条目能够推动人物行动或剧情冲突。
- [ ] 确认、建议和推测已区分。

## 9. 安全边界

- 不创建无限能力或无代价资源。
- 不覆盖无关世界书条目。
- 用户确认前不写入正式目录。
