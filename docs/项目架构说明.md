# Storydex 项目架构说明

本文档用于快速理解 Storydex 开源包的主要目录、运行链路和数据边界。

## 总体架构

Storydex 是一个本地优先的桌面写作工作台，由三部分组成：

- `apps/frontend`：Vue 3 + TypeScript 工作台界面。
- `apps/backend`：FastAPI 本地服务，负责文件、项目、Agent、版本控制、预设、知识图谱和使用指南接口。
- `apps/desktop`：Electron 桌面壳，负责桌面窗口、本地服务协调和应用资源同步。

常见运行方式是：启动脚本准备 Python 与 Node 依赖，拉起后端服务和前端服务，桌面端加载前端工作台。前端通过 HTTP API 调用后端，后端再访问用户选择的小说项目目录。

## 开源目录

```text
Storydex/
├─ apps/
│  ├─ frontend/          # Vue 工作台
│  ├─ backend/           # FastAPI 后端服务
│  └─ desktop/           # Electron 桌面壳
├─ assets/               # 项目 LOGO、吉祥物、组织 LOGO 和应用图标
├─ docs/
│  ├─ guide/             # 内置使用指南
│  ├─ prompts/           # 指令仓库 Markdown 模板
│  ├─ skills/            # 小说项目内置 Agent 技能模板
│  ├─ assets/readme/     # README 展示图片
│  └─ 项目架构说明.md
├─ scripts/              # 启动所需脚本
├─ start-storydex.bat    # 一键启动前后端开发环境
├─ start-desktop.bat     # 启动桌面开发环境
├─ start-desktop.ps1     # PowerShell 桌面启动入口
├─ README.md
├─ LICENSE
├─ COMMERCIAL-LICENSE.md
└─ requirements.txt
```

## 前端

`apps/frontend` 提供 Storydex 主工作台：

- `src/layouts/WorkbenchLayout.vue`：资源栏、编辑区、Coomi 面板的主布局。
- `src/components`：文件树、编辑器、Agent 面板、版本控制、预设管理、系统设置等组件。
- `src/stores`：工作区、Agent、认证、主题和 UI 状态。
- `src/api`：后端 API 客户端。
- `src/assets/theme.css`：主题、布局和基础视觉样式。

前端只处理界面和交互状态，不直接越过后端访问用户工作区。

## 后端

`apps/backend` 提供本地 API 与项目服务：

- `main.py`：FastAPI 应用入口和路由注册。
- `api/`：HTTP 路由层。
- `services/`：项目文件、Agent 编排、版本控制、知识图谱、使用指南等业务逻辑。
- `core/`：配置、日志、异常和通用基础能力。
- `storage/`：本地工作区访问适配。

后端负责路径归一化和项目边界控制，避免 Agent 或前端任意读写工作区之外的文件。

## 桌面端

`apps/desktop` 提供 Electron 桌面开发与打包能力：

- `electron/main.cjs`：Electron 主进程。
- `electron/preload.cjs`：预加载脚本。
- `scripts/sync-app-assets.cjs`：同步前端构建产物与应用资源。
- `package.json`：桌面端开发、构建和打包命令。

桌面端会优先使用应用内置资源，并在开发模式下协调前端与后端服务。

## Storydex 小说项目

用户创建或打开的是一个小说项目目录。典型结构如下：

```text
your-novel-project/
├─ chapters/             # 正文与章节
├─ .gitignore            # 默认忽略 Storydex Agent 运行态
└─ .storydex/
   ├─ project.json       # 项目元数据
   ├─ config/            # 项目配置
   ├─ presets/           # 写作预设
   ├─ characters/        # 角色资产
   ├─ worldbook/         # 世界观资料
   ├─ memory/            # 创作记忆与章节滚动摘要
   ├─ wiki/              # 知识图谱 / LLM WIKI
   ├─ .cache/            # 全文检索索引等本地缓存，可随时删除重建
   └─ .agent/
      ├─ sessions/       # Agent 会话历史
      ├─ plans/          # Agent 计划
      └─ temp/           # Agent 临时文件
```

真实小说项目可能包含用户创作内容、模型输出、本地状态和私有配置，不应提交到 Storydex 源码仓库。

## 发布边界

开源仓库保留运行 Storydex 所需的主要源码、启动入口、说明文档和展示资源。以下内容不应提交：

- `.env`、`.env.*`、API Key、私钥、证书、本地账号配置。
- `node_modules`、`.python39`、虚拟环境、构建产物、缓存和日志。
- 用户小说项目、临时工作区、测试工作区、发布包和调试输出。
- 内部实验、一次性验证脚本、工具日志和私有评测材料。

根目录 `.gitignore` 会覆盖这些敏感与运行态文件；公开发布前仍应检查 `git status --short`，确认没有误加入本地私有文件。
