# Storydex 项目架构说明

本文档概述 Storydex 的主要目录、运行链路和模块边界，用于开源仓库的快速阅读与二次评估。

## 总体架构

Storydex 采用本地优先的桌面应用架构：

- `apps/frontend` 提供 Vue 3 工作台界面。
- `apps/backend` 提供 FastAPI 本地服务，负责项目文件、Agent 编排、版本控制、预设、知识图谱和系统配置。
- `apps/desktop` 提供 Electron 桌面壳，负责窗口、菜单、本地进程启动和桌面打包。
- `docs/使用指南` 提供面向用户的功能说明。
- `scripts` 保留启动和构建所需脚本。

运行时通常由 Electron 打开前端工作台，并由桌面端或启动脚本拉起本地后端服务。前端通过 HTTP API 与后端通信，后端再访问本地小说项目目录、`.storydex` 元数据和 Git 工作区。

## 前端工作台

前端位于 `apps/frontend`，核心职责包括：

- 渲染写作工作台、文件树、编辑器、Agent 面板、版本控制面板和系统设置窗口。
- 通过 `src/api` 下的客户端模块调用后端接口。
- 通过 Pinia store 管理工作区、Agent 会话、主题、预设和认证状态。
- 使用 `src/assets/theme.css` 和主题常量提供多套界面外观。

主要入口：

- `src/main.ts`：前端应用入口。
- `src/App.vue`：应用根组件。
- `src/layouts/WorkbenchLayout.vue`：工作台布局。
- `src/components`：编辑器、侧边栏、Agent 面板、设置窗口等 UI 组件。
- `src/stores`：状态管理。
- `src/api`：后端接口封装。

## 后端服务

后端位于 `apps/backend`，核心职责包括：

- 提供 FastAPI HTTP 接口。
- 管理 Storydex 项目文件与 `.storydex` 元数据。
- 执行 Agent 编排、工具调用、上下文组装和运行轨迹记录。
- 管理 Git 快照、文件历史、预设、知识图谱和系统配置。
- 对本地文件访问进行边界控制和路径归一化。

主要入口：

- `main.py`：FastAPI 应用入口与路由注册。
- `api/`：HTTP 路由层。
- `services/`：业务服务层。
- `core/`：配置、数据库、异常、日志和通用基础能力。
- `storage/`：工作区文件访问适配。
- `tests/`：后端单元测试与集成测试。

## 桌面端

桌面端位于 `apps/desktop`，核心职责包括：

- 启动 Electron 窗口并加载前端页面。
- 在桌面开发或打包场景中协调前端、后端和应用资源。
- 提供 Windows 打包配置与图标资源同步。

主要入口：

- `electron/main.cjs`：Electron 主进程。
- `electron/preload.cjs`：预加载脚本。
- `scripts/sync-app-assets.cjs`：桌面资源同步。
- `package.json`：开发、构建和打包命令。

## 数据与项目模型

Storydex 以用户选择的小说项目目录作为工作区。一个典型小说项目包含：

```text
your-novel-project/
├─ chapters/             # 正文与章节文件
└─ .storydex/            # Storydex 项目元数据
   ├─ config/            # 项目设置
   ├─ presets/           # 写作预设与编译结果
   ├─ characters/        # 角色资产
   ├─ worldbook/         # 世界观资料
   ├─ memory/            # 记忆与状态
   ├─ wiki/              # 知识图谱 / LLM WIKI
   ├─ trace/             # Agent 运行轨迹
   └─ .agent/            # Agent 会话、计划和临时资产
```

`.storydex` 下的运行态文件可能包含用户创作内容、模型输出、配置和本地状态。公开仓库不应提交真实用户工作区、密钥、日志或运行缓存。

## 运行链路

1. 用户执行 `start-storydex.bat` 或开发脚本。
2. 脚本安装或校验必要环境，并启动前端、后端或 Electron。
3. Electron 加载前端工作台。
4. 前端通过 API 请求后端读取项目、保存文件、运行 Agent 或生成知识图谱。
5. 后端在本地工作区执行文件、Git、预设和 Agent 编排操作。
6. 前端展示编辑结果、工具调用、版本状态和运行轨迹。

## 开源精简边界

开源仓库应保留项目运行与阅读所需的主要目录：

- `apps/frontend`
- `apps/backend`
- `apps/desktop`
- `assets`
- `docs/使用指南`
- `docs/assets/readme`
- `scripts` 中的启动和构建脚本
- 根目录启动脚本、依赖声明、许可证和 README

不建议公开提交以下内容：

- `.env`、`.env.*`、API key、私钥、证书和本地账号配置。
- `node_modules`、`.venv`、`.python39`、构建产物、缓存和日志。
- 临时工作区、测试工作区、用户小说项目、发布包和调试输出。
- 与公开运行无关的内部实验数据、一次性验证脚本和工具日志。
