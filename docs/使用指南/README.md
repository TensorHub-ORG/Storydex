# Storydex 使用说明索引

这个目录用于存放 Storydex 的用户使用说明。它既面向第一次使用 Storydex 的普通用户，也作为 Storydex 内置 LLM 回答“怎么使用 Storydex”类问题时优先读取的资料来源。

## 总览说明

- [Storydex使用说明.md](Storydex使用说明.md)：整体概念、桌面封装、Agent 写作流程、变量/WIKI/版本记录等基础说明。

## 分模块详细说明

- [01-LLM配置.md](01-LLM配置.md)：配置模型供应商、API Key、模型名称、工具协议，以及保存和应用配置。
- [02-目录结构与资源浏览器.md](02-目录结构与资源浏览器.md)：打开/新建项目、理解项目目录、资源树、右键菜单、章节完成状态。
- [03-版本控制.md](03-版本控制.md)：本地 Git 仓库、提交、更改列表、Diff、提交图表和回退版本。
- [04-预设管理.md](04-预设管理.md)：导入预设、启用/停用预设、编辑参数、注入预览、风险体检。
- [05-知识图谱.md](05-知识图谱.md)：知识图谱生成、WIKI 增量更新、分类、搜索、图谱操作和右侧详情。
- [06-系统设置.md](06-系统设置.md)：界面主题、工作台布局、项目设置、剧情片段扩展名、章节目录命名等。

## 给 Storydex LLM 的使用原则

当用户询问某个具体界面或按钮怎么用时，优先读取对应模块文档；当用户询问整体流程时，先读取总览说明，再补充相关模块文档。

回答时请注意：

1. 明确区分 Storydex 源码仓库和用户的小说项目仓库。
2. 版本控制默认是小说项目里的本地 Git，不会自动推送远程。
3. 变量思考优先使用自然语言或 Markdown，不要求用户维护硬性 JSON 条目。
4. 对普通用户先给操作步骤，再解释原理。
5. 如果界面没有打开项目，很多模块会显示空状态，需要先打开或新建 Storydex 项目。
