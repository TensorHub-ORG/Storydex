# 知识图谱详细说明

知识图谱用于把 Storydex 小说项目中的角色、关系、剧情、设定和项目概览可视化。它是故事事实的检索和展示层，不替代角色文件、变量文件、正文和 WIKI。

知识图谱可以由 LLM 深度生成，也可以在 LLM 不可用时使用本地解析结果进行基础展示。

## 入口

1. 打开 Storydex 小说项目。
2. 点击左侧活动栏的知识图谱图标。
3. 打开 `知识图谱` 面板。

## 界面区域

| 区域 | 说明 |
| --- | --- |
| 顶部标题 `知识图谱` | 当前模块名称。 |
| 顶部操作按钮 | 生成图谱、更新 WIKI、审阅 WIKI、刷新。 |
| 分类标签 | 在角色关系、角色档案、剧情、设定、项目概览之间切换。 |
| 搜索框 | 搜索条目、节点或关系。 |
| 中间图谱画布 | 展示节点和关系。 |
| 右侧详情面板 | 展示项目概览、节点详情、关系证据和相关条目。 |
| 底部缩放控件 | 缩小、放大、适配视图。 |

## 顶部按钮

| 图标/按钮 | 标题 | 作用 | 适合场景 |
| --- | --- | --- | --- |
| `auto_awesome` | `Agent 深度生成知识图谱` | 调用 Agent 对项目进行更深入的知识图谱生成。 | 新项目导入后、章节大量变化后、需要更完整关系网络时。 |
| `published_with_changes` | `Agent 增量更新 WIKI` | 根据最近变化增量更新 WIKI。 | 生成新剧情、角色关系变化、变量变化后。 |
| `fact_check` | `Agent 审阅 WIKI` | 审阅 WIKI 条目是否准确、一致、可维护。 | WIKI 变多、设定冲突、角色信息不一致时。 |
| `refresh` | `刷新` | 重新读取当前知识图谱和 WIKI 数据。 | 手动改文件或 Agent 执行结束后。 |

## 分类标签

| 标签 | 作用 |
| --- | --- |
| `角色关系` | 重点展示角色之间的关系网络。 |
| `角色档案` | 查看角色条目、角色摘要和来源文件。 |
| `剧情` | 查看事件、章节、剧情节点。 |
| `设定` | 查看世界观、地点、势力、规则等设定条目。 |
| `项目概览` | 查看项目整体摘要、来源文件数量、图谱构建信息。 |

标签后面的数字表示当前分类下可用条目数量。

## 搜索与统计

| 元素 | 作用 |
| --- | --- |
| `搜索条目、节点或关系` | 输入角色名、地点、势力、事件关键词。 |
| 清除按钮 | 清空当前搜索。 |
| `节点 · 连接` | 显示当前图谱节点和关系数量。 |
| `待确认` | 表示有条目可能需要用户确认或审阅。 |

搜索适合快速定位某个角色、关系或设定来源。

| 操作 | 结果 |
| --- | --- |
| 鼠标滚轮 | 放大或缩小图谱。 |
| 拖动画布空白处 | 平移视图。 |
| 点击节点 | 在右侧详情面板查看节点信息。 |
| 拖动节点 | 调整节点位置，方便观察关系。 |
| 点击连线 | 查看两端节点的关系、强度和证据。 |
| 点击图例 | 理解不同颜色或类型节点的含义。 |

## 底部缩放按钮

| 按钮 | 作用 |
| --- | --- |
| `-` / `缩小` | 缩小图谱。 |
| 百分比 | 当前缩放比例。 |
| `+` / `放大` | 放大图谱。 |
| `fit_screen` / `适配视图` | 自动调整视图，让图谱尽量完整显示。 |

如果图谱节点跑到视野外，优先点击 `适配视图`。

右侧详情会根据当前选择变化。

### 项目概览

| 内容 | 说明 |
| --- | --- |
| 项目名称 | 当前小说项目名。 |
| 生成方式 | 例如 LLM 生成、本地推断、local fallback。 |
| 更新时间 | 图谱或 WIKI 最近更新时间。 |
| 项目总览 | 对项目内容的摘要。 |
| 工作区路径 | 当前项目本地路径。 |
| 章节/正文文件数量 | 被图谱读取到的正文来源数量。 |
| 识别角色数量 | 当前识别到的候选角色数量。 |
| 索引来源文件 | 被用于构建图谱的文件数量。 |
| 来源路径 | 支撑该概览的项目文件路径。 |

### 节点详情

| 内容 | 说明 |
| --- | --- |
| 类型 | 角色、地点、势力、事件、设定等。 |
| 标题 | 节点名称。 |
| 审阅状态 | 是否需要确认、是否已审阅。 |
| 来源标签 | 该信息来自正文、角色文件、WIKI 或本地推断。 |
| 摘要 | 节点的简要说明。 |
| 细节 | 更具体的事实或设定。 |
| 来源路径 | 支撑该节点的文件路径。 |
| 相关条目 | 和该节点关联的其他条目，可点击跳转。 |

### 关系详情

| 内容 | 说明 |
| --- | --- |
| 关系类型 | 角色关系、章节共现、势力关联等。 |
| 两端节点 | 关系连接的两个对象。 |

| 证据 | 支撑该关系的章节、描述或 WIKI 片段。 |

## 常用操作

### 第一次生成知识图谱

1. 打开小说项目。
2. 进入 `知识图谱`。
3. 点击 `Agent 深度生成知识图谱`。
4. 等待 Agent 读取项目文件并生成图谱。
5. 生成后点击 `刷新`。
6. 在 `项目概览` 查看来源文件和统计数量。

### 剧情生成后更新 WIKI

1. 完成新剧情生成。
2. 确认角色、物品、变量、事实和关系增量。
3. 进入 `知识图谱`。
4. 点击 `Agent 增量更新 WIKI`。
5. 等待 Agent 更新 WIKI。
6. 点击 `刷新` 查看图谱变化。

如果开启了自动更新 WIKI，Storydex 可以在变量更新后自动执行，但仍建议定期审阅。

### 审阅 WIKI

1. 点击 `Agent 审阅 WIKI`。
2. 让 Agent 检查条目是否重复、冲突或过期。
3. 根据结果确认需要更新的 WIKI 文件。
4. 查看版本控制中的 Diff。

## 知识图谱和 WIKI 的关系

| 项目 | 作用 |
| --- | --- |
| 角色文件 | 存放角色档案，是角色事实的重要来源。 |
| 变量文件 | 存放当前故事状态、关系、物品归属、旗标等。 |
| WIKI | 面向阅读和检索的事实条目。 |
| 知识图谱 | 把角色、事件、设定和关系可视化。 |

知识图谱不应该替代角色文件和变量文件。它更像“地图”和“检索层”。

## 注意事项

- 图谱中的关系可能来自推断，重要设定仍建议写回角色文件、变量或 WIKI。
- `local fallback` 表示 LLM 不可用时的本地构建结果，质量可能不如 Agent 深度生成。
- 自动更新 WIKI 会增加耗时；和自动更新变量一起使用时，建议每次只生成单条剧情片段。
- 如果图谱节点太密集，可以切换分类、搜索关键词或点击适配视图。

## 可以这样问 Agent

```text
请根据当前项目深度生成知识图谱，重点整理角色关系、势力关系和关键事件。
```

```text
请增量更新 WIKI，只处理最近生成剧情中新增或变化的角色、地点、物品和关系。
```

```text
请审阅当前 WIKI，找出重复条目、设定冲突和来源不清的内容。
```

```text
请解释知识图谱里“青萝”和“秘密”的关系证据来自哪里。
```

## 常见问题

### 为什么图谱里角色很多但关系很少？

可能是项目文件里角色出现了，但缺少明确互动或关系描述。可以让 Agent 深度生成图谱，或先整理角色文件和最近章节。

### 为什么右侧显示 local fallback？

表示当前图谱是本地解析构建的基础结果。通常是 LLM 不可用、未配置或生成失败时出现。

### 图谱中的内容一定准确吗？

不一定。图谱可能包含模型推断或本地解析结果。重要事实应以正文、角色文件、变量文件和 WIKI 为准。

### 更新 WIKI 会直接改文件吗？

如果 Agent 具备修改权限，更新 WIKI 会写入小说项目文件。执行结束后若有修改，应产生本地 Git 提交，方便追踪。
