# 系统设置详细说明

系统设置用于调整 Storydex 的界面主题、工作台布局和项目级写作设置。

系统设置分为两类：

| 类型 | 保存位置 | 作用 |
| --- | --- | --- |
| 界面与布局设置 | 本机用户配置 | 影响 Storydex 在当前电脑上的显示和布局。 |
| 项目设置 | 小说项目 `.storydex/config/project-settings.json` | 影响当前小说项目的章节、剧情片段和 Agent 提交行为。 |

## 入口

1. 点击左侧活动栏底部的设置图标。
2. 在设置菜单中选择系统设置。
3. 打开 `系统设置` 窗口。

系统设置是弹窗形式。可以点击右上角关闭，也可以点击弹窗外部关闭。

## 界面区域

| 区域 | 说明 |
| --- | --- |
| 左侧标题 `系统设置` | 当前窗口名称。 |
| 搜索框 | 搜索设置分类，例如主题、章节、布局。 |
| 左侧分类导航 | 切换不同设置分类。 |
| 右侧设置卡片 | 编辑当前分类的具体设置。 |
| 卡片右上角按钮 | 重新加载、保存、关闭。 |

## 左侧搜索框

| 元素 | 作用 |
| --- | --- |
| `搜索设置（主题、章节、布局）` | 输入关键词过滤设置分类。 |
| 搜索结果提示 | 显示找到几个匹配分类，或提示没有匹配分类。 |

常用关键词：

```text
主题
字体
布局
资源栏
Agent
章节
剧情片段
```

## 左侧分类

| 分类 | 作用 |
| --- | --- |
| `界面与主题` | 设置界面主题、文件字体大小、玩家模式字体大小。 |
| `工作台布局` | 设置资源栏宽度、Agent 栏宽度。 |
| `项目设置` | 设置当前小说项目的剧情片段格式、每章片段上限、章节目录命名和 Agent 提交提示。 |

## 卡片右上角按钮

| 图标/按钮 | 标题 | 作用 |
| --- | --- | --- |
| `refresh` | `重新加载` | 放弃当前未保存编辑，重新读取该分类设置。 |
| `save` | `保存` | 保存当前分类设置。 |
| `close` | `关闭` | 关闭系统设置窗口。 |

界面与布局通常会自动保存；项目设置建议修改后点击 `保存`。

## 界面与主题

### 界面主题

主题区域显示多个主题卡片。

| 操作 | 结果 |
| --- | --- |
| 点击主题卡片 | 切换 Storydex 界面主题。 |
| 查看主题预览色块 | 快速判断主题风格。 |

主题设置会影响整个应用外观，不影响小说项目内容。

### 字体大小

| 设置项 | 范围 | 作用 |
| --- | --- | --- |
| `文件显示字体大小` | 12-24 px | 调整中间文件编辑/预览区域的字体大小。 |
| `玩家模式字体大小` | 12-28 px | 调整玩家模式或阅读模式的字体大小。 |

拖动滑块即可调整。界面会显示当前像素值。

## 工作台布局

| 设置项 | 范围 | 作用 |
| --- | --- | --- |
| `资源栏宽度` | 220-520 px | 调整左侧资源浏览器等侧栏宽度。 |
| `Agent 栏宽度` | 320-760 px | 调整右侧 Agent 面板宽度。 |

如果资源树文件名显示不完整，可以加宽资源栏。如果 Agent 对话和任务浮条拥挤，可以加宽 Agent 栏。

## 项目设置

项目设置必须先打开小说项目才可用。保存路径通常为：

```text
.storydex/config/project-settings.json
```

顶部信息条会显示：

| 项目 | 说明 |
| --- | --- |
| `来源` | 当前设置来源，例如 API 优先或本地文件。 |
| `更新` | 设置最近更新时间。 |
| `路径` | 当前项目设置文件路径。 |

### 剧情片段扩展名

| 选项 | 作用 |
| --- | --- |
| `Markdown .md` | 新剧情片段使用 Markdown 文件。推荐默认使用。 |
| `Text .txt` | 新剧情片段使用纯文本文件。 |

如果需要标题、列表、注释等结构，推荐 `.md`。

### 每章剧情片段上限

| 设置 | 说明 |
| --- | --- |
| `每章剧情片段上限` | 控制每个章节目录最多容纳多少个剧情片段。 |
| 范围 | 1-99。 |

这个设置用于约束章节目录结构。截图中的示例值为 3，表示每章最多 3 个剧情片段。

### 自动命名章节目录

| 状态 | 含义 |
| --- | --- |
| 开启 | Storydex 自动按规则命名章节目录，兼容历史项目。 |
| 关闭 | 用户或模板更明确地控制章节目录名称。 |

如果旧项目依赖自动命名，可以保持开启。新项目如果有明确章节模板，可以按模板管理。

### Agent 结束后询问提交

| 状态 | 含义 |
| --- | --- |
| 开启 | 检测到小说项目有未提交修改时弹出提交确认。 |
| 关闭 | Agent 执行结束后按系统规则自动处理本地提交，不额外询问。 |

如果 Storydex 已按“Agent 执行结束且有修改就自动提交”的规则运行，可以关闭询问，减少打断。

## 常用操作

### 调整主题

1. 打开 `系统设置`。
2. 进入 `界面与主题`。
3. 点击喜欢的主题卡片。
4. 观察界面是否切换成功。

### 调整编辑器字体

1. 进入 `界面与主题`。
2. 拖动 `文件显示字体大小`。
3. 打开一个正文文件检查阅读舒适度。

### 调整侧栏宽度

1. 进入 `工作台布局`。
2. 调整 `资源栏宽度` 或 `Agent 栏宽度`。
3. 观察左右面板是否适合当前屏幕。

### 修改剧情片段格式

1. 打开小说项目。
2. 进入 `系统设置` -> `项目设置`。
3. 在 `剧情片段扩展名` 中选择 `.md` 或 `.txt`。
4. 点击 `保存`。
5. 后续新生成片段会按该格式创建。

### 修改每章片段上限

1. 打开 `项目设置`。
2. 修改 `每章剧情片段上限`。
3. 点击 `保存`。
4. 后续章节组织会参考新上限。

## 和 Agent 剧情生成设置的关系

系统设置里的 `每章剧情片段上限` 是项目结构规则；Agent 面板里的 `片段数量` 是本轮生成数量。

例如：

| 设置 | 含义 |
| --- | --- |
| 每章剧情片段上限 = 3 | 一个章节目录最多放 3 个片段。 |
| 本轮片段数量 = 1 | 这次只生成 1 个片段。 |
| 本轮片段字数 = 2000 | 这次每个片段目标约 2000 字。 |

如果开启自动更新变量或自动更新 WIKI，建议本轮片段数量保持 1，因为自动更新变量需要较多耗时。

## 注意事项

- 界面和布局设置主要影响当前电脑，不改变小说内容。
- 项目设置会写入当前小说项目，可能进入本地 Git 版本记录。
- 修改项目设置后，如果看到版本控制出现 `project-settings.json` 变化，这是正常现象。
- 没有打开项目时，项目设置不可编辑。
- 新手用户优先只改主题、字体和 Agent 栏宽度，项目结构设置可以保持默认。

## 常见问题

### 为什么项目设置打不开？

需要先打开一个 Storydex 小说项目。没有项目时，系统设置只能调整界面和布局。

### 调整字体会影响导出的小说吗？

不会。字体大小只影响 Storydex 界面显示，不修改正文文件内容。

### 每章片段上限会删除多余片段吗？

不应该。它是组织规则，不应自动删除已有正文。已有内容应通过目录整理或 Agent 规划处理。

### 改项目设置后为什么版本记录有变化？

项目设置写入 `.storydex/config/project-settings.json`，属于小说项目文件。Storydex 记录这类变化是为了保证可追溯。
